Aula 04 - Criando BD e usando constraint

Usando constraint na criação do BD

Constraint são os parâmetros que são enviados após o comando básico de criação de banco.

No exemplo, vamos usar constraint para criar um banco com codificação UTF-8, dessa forma, ele irá entender os caracteres acentuados.

        
            CREATE DATABASE cadastro DEFAULT CHARACTER SET utf8 DEFAULT COLLATE utf8_general_ci;

Criação de tabelas usando constraint

        
            create table pessoas (
            nome varchar(30) not null,
            nascimento date,
            sexo enum('M', 'F'),
            peso decimal (5,2),
            altura decimal (3,2),
            nacionalidade varchar(20) default 'brasil'
            )  default charset = utf8;

Com o comando abaixo, cria-se a mesma tabelas, mas usando a constraint primary key que serve para não repetir valores. 

        
            create table pessoas (
            id int not null auto_increment,
            nome varchar(30) not null,
            nascimento date,
            sexo enum('M', 'F'),
            peso decimal (5,2),
            altura decimal (3,2),
            nacionalidade varchar(20) default 'brasil',
            primary key(id)
            )  default charset = utf8;
